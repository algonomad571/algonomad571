## ⚡ About

Engineer focused on the intersection of **high-throughput systems**, **distributed architecture**, and **on-chain finance**. I work in the parts of the stack where correctness guarantees meet performance constraints — lock-free queues, concurrency-safe transaction layers, MEV-aware execution engines.

Currently building systems in **C++**, **Go**, and **Node.js** that operate at sub-millisecond latency or handle thousands of concurrent connections. My work spans from kernel-adjacent async I/O (Boost.Asio) all the way to EVM-level transaction sequencing.

Not interested in CRUD. Interested in systems that **can't afford to be slow**.

- 🔬 **AI Agent Evaluation Intern** @ AirDawg Labs — adversarial testing, reproducibility infra, Docker-based eval pipelines
- 🌱 **GirlScript Summer of Code** — Top 100 of 2000+, 12+ PRs merged across WebSockets and CI/CD tooling
- 🏆 **Codeforces Expert** — ~1675 rating, consistent problem setter in algorithmic design

## 🚀 Featured Projects

### [`Market Pulse`](https://github.com/algonomad571) — Ultra-Low-Latency Market Data Ingestion Engine
> Real-time financial data pipeline built for maximum throughput and minimum jitter.

- **150,000+ messages/sec** sustained ingestion using **Boost.Asio** async I/O
- **Lock-free queue architecture** — zero mutex contention on the hot path
- Cache-aligned memory layouts, branch-prediction-optimized dispatch, sub-millisecond P99 latency

---

### [`BoltPay`](https://github.com/algonomad571) — Distributed Payment Processing System
> Horizontally scalable payment infrastructure built for fault-tolerance and operational visibility.

- Handles **10,000+ req/sec** with idempotency guarantees and exactly-once semantics
- Distributed saga orchestration for cross-service transaction consistency
- Full observability stack: structured logging, tracing, alerting, and metrics



### [`Rogi Mitra`](https://github.com/algonomad571) — Hospital Management System
> QR-based patient management platform for multi-department hospital workflows.

- Role-based access control (RBAC) with scoped JWT authorization per user type
- **Concurrency-safe appointment booking** — optimistic locking to eliminate double-booking races
- QR-linked patient records enabling instant context retrieval across departments
